# v2 への引き継ぎポイント

v1.0の凍結・実運用準備フェーズで得られた知見を、次期バージョン（v2）検討時のために記録する。

## 1. 積み残しの明確な課題

- **既知バグA**: `line_response_rate`実データ接続（`docs/V1_0_KNOWN_ISSUES.md`）
- **既知バグB**: 音声メモ許可タイムアウトのタイマーリーク（同上）
- **「VIP管理」ボタン**: 設定タブの`MenuDashboard.tsx`に残存。削除かラベル変更が必要
  （`docs/V1_0_FINAL_READINESS_REPORT.md` 3章）
- **LINE領域の評価系表示**: `LineApprovalScreen.tsx`／`ChatWindow.tsx`に「優先度◯点」
  「VIP候補」「離脱リスク検知」等の表示が残る。v1.0では意図的にスコープ外としたが、
  現場スタッフの通常導線（今日タブ→LINE未読アイコン）から到達可能なため、v2では
  正式にスコープに含めて是正すべき

## 2. 顧客詳細シートの情報密度

`CustomerBottomSheet.tsx`は、評価系UIを排除した後も依然としてパネル数が多い
（会話メモ・肌タグ・AI提案・TL-5・接客リスク・成功パターン・ホームケア・LINE下書き等）。
「30秒で状況を把握する」という設計思想を顧客詳細シート自体にも及ぼすなら、
パネルの優先順位付け・段階的開示（今は今日タブのブリーフィングのみが担っている役割）を
検討する余地がある。

## 3. データ基盤の整備

- 口コミ件数（わたしタブの4指標目）は現行DBにテーブルが存在せず、v1.0では
  「準備中」表示で対応した。実装するには口コミデータソース（Google/LINE連携か手入力）の
  設計が別途必要
- LINE反応率（既知バグA）と合わせ、「事実情報」として表示している値の中に、
  実際にはプレースホルダーや未接続の値が混在していないか、v2着手時に棚卸しすることを推奨

## 4. 内部エンジンとUI表示の分離という設計方針の継続

v1.0では「churn_risk・vip_rankなどの内部スコアリング値は、NextActionPanel・CustomerRiskCard・
adaptivePriorityなど内部の優先度判定にのみ使用し、画面には一切数値を出さない」という方針で
整理した（`CustomerRiskCard.tsx`のコメント「数値を直接見せない。自然言語で状態を表現。」が
象徴的）。v2で新しい画面や機能を追加する際も、この分離を維持することを推奨する。新機能の
UIレビュー時は「内部で使っているスコアが、画面のどこかに数値・記号（★等）として漏れ出て
いないか」を確認項目に加えるとよい。

## 5. 運用ルールの資産

`CLAUDE.md`・`docs/V1_FREEZE_SAFETY_RULES.md`に、開発エージェントの安全制御ルール
（スコープ外変更の禁止・無応答時の先行実行禁止・未承認ロールバックの禁止等）を整備した。
v2開発時もこの運用ルールを土台として継続利用することを推奨する。
