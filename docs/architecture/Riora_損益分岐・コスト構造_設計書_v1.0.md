## 1. コスト分類(固定費 / 変動費)の確定

## 2. 社会保険料(ハイブリッド方式・報酬総額ベース・確定)

**重要**: 指名バック(¥250/件)も歩合(5%)も**報酬の一部=社会保険の対象**。よって概算は固定給だけでなく当月の歩合・指名込みの**報酬総額**に係数をかける。

```
社保対象給与(当月・実績連動) =
  鈴木 ¥450,000(役員報酬・歩合なし)
  + 亀山 (固定給¥250,000 + 当月歩合 + 当月指名バック)
  + 外舘 (固定給¥220,000 + 当月歩合 + 当月指名バック)
  ※岸の役員報酬・久保田の外注費は社保対象外

常時表示(概算): 社保対象給与 × 概算係数15.5%(会社負担分)
  例(標準的な月・亀山外舘の歩合+指名が各 月¥4〜5万のとき):
    対象給与 ≒ ¥1,000,000 前後 → 概算 ≒ ¥150,000〜155,000

  ※固定給ベースの旧概算(¥920,000×15%=¥138,000)より月1万円強 上振れする。
    これが「歩合・指名込み」の実態。固定値でなく当月実績で再計算する。

月1回上書き(実額): freee人事労務の会社負担社会保険料の確定額を
  business_settings.social_insurance_actual に手入力 → 概算を上書き(最も正確)
表示: 経営TOPで「概算(実績連動)」か「実額(freee反映済)」かをラベルで区別
```

設計判断: 概算係数は**15.5%**(健康保険+厚生年金+労働保険の会社負担分のおおよそ。料率は地域・年度で変動するため business_settings で調整可能)。社保対象給与は**夜間バッチで当月売上が確定するたび再計算**(固定値で持たない)。**Riora側で等級表・標準報酬月額・折半の厳密計算は持たない**(freeeが正)。概算はあくまで損益分岐の速報用で、確定はfreee実額の手入力で補正する。

## 3. 損益分岐点の計算式(実数)

```
変動費率 = (インセンティブ率5% + Square率2.5%×キャッシュレス比率 + 物販原価率×物販比率)
  ※指名バックは件数依存のため、簡易版では「指名バック総額」を固定費的に月次実績で扱うか、
    変動費率に平均指名率を織り込む。下記は簡易版(指名バックを変動費総額に合算)。

簡易版(まず動かす):
  変動費率 ≈ 5%(歩合) + 2.5%(Square・全売上がキャッシュレスと仮定) ≈ 7.5%
  → 物販原価・指名バックは月次実績額として変動費に加算

損益分岐点売上 = 固定費 ÷ (1 − 変動費率)
  = ¥1,715,446 ÷ (1 − 0.075)
  = ¥1,715,446 ÷ 0.925
  ≈ 月商 ¥1,854,536

→ 光熱費・通信費・消耗品(実績入力)を仮に月¥80,000とすると固定費¥1,795,446、
   損益分岐 ≈ 月商 ¥1,940,482
```

**現在地の目安: 月商おおよそ185〜194万円を超えると黒字。** 客単価¥28,000なら月66〜70名、3名スタッフで射程内。

注: 社保は報酬総額ベースのため、売上が伸びる(=歩合・指名が増える)と社保も微増し固定費がわずかに上がる。厳密には社保だけ準変動的だが、損益分岐の速報用途では当月実績の社保額を固定費に算入すれば十分正確。

精密版(Phase2): キャッシュレス比率・指名率・物販比率を実績から算出し変動費率を動的計算。初期は簡易版で十分(光熱費等の変動の方が誤差として大きいため)。

## 6. freee との関係(二重計算しない)

| 項目 | Riora の役割 | freee の役割(正) |
|---|---|---|
| 給与の手取り・社保の1円計算 | 持たない | **正**(人事労務) |
| 役員報酬・固定給・交通費の額 | 損益分岐用に保持(概算) | 確定値の源泉 |
| 会社負担社保 | 概算表示+月1実額上書き | 実額の源泉 |
| 売上・経費の会計帳簿 | 持たない | **正**(会計) |
| 損益分岐・着地予測・現在地 | **正**(経営の意思決定ツール) | 持たない |

役割分担: **freee=確定した過去の正確な帳簿 / Riora=今この瞬間の経営判断**。Rioraは「あと何人で黒字か」を毎日見るための速報、freeeは月次の確定値。両者は社保実額の手入力1点で繋がる(API連携はPhase2で検討)。
