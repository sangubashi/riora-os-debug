**株式会社martylabo / Salon Riora**
作成日: 2026-06-12
位置づけ: v1.0への修正指示を反映した差分改版。本書とv1.0が矛盾する場合は本書を正とする。
改版理由: Riora思想の明文化 — **「スタッフに必要最小限の数字だけ見せる」**

## 1. 完全削除(スタッフが見る全画面から)

以下はスタッフ向けのいかなる画面・通知・キャラクターメッセージにも表示しない:

| 削除対象 | 備考 |
|---|---|
| サブスク目標 / 口コミ目標 / ブログ目標 / 継続率目標 / 店舗利益目標 | 目標として設定すること自体を廃止(管理者画面にも個人別では持たない) |
| スタッフランキング / 売上ランキング / 口コミランキング / ブログランキング | v1.0で禁止済み。本版で「スタッフ自身の画面にも比較情報を出さない」まで拡張 |

実装レベルの強制: スタッフ向けAPIレスポンスの型に上記フィールドを**定義しない**(削除ではなく不存在)。受入テストでレスポンスJSON全走査(ranking/rank/target系の禁止キー検査+他スタッフのstaff_id不出現検査)。

```typescript
export interface StaffMyKpiResponse {
```

| 項目 | データソース |

## 3. Brain学習専用データ(スタッフ非表示の確定リスト)

以下は学習・管理者分析専用。スタッフ画面・ブリーフィング内の自己成績文脈・キャラクターメッセージのいずれにも出さない:

```
継続率 / 再来率 / 提案成功率(受諾率) / サブスク化率 / LINE反応率 /
visit_score / CSI / パターン活用率 / churn関連 / uplift / 入力完了率(個人別)
```

境界の明確化:
- **ブリーフィングは引き続き表示する**(「今日の顧客に何を提案するか」は数字ではなく行動指示。successReferenceの「同型顧客で72%」は**顧客セルの統計であり本人の成績ではない**ため存続)
- 管理者画面(③Staff Strength Analytics)は**Managerのみ**閲覧可のまま存続(強み発見・移転目的・ランキングなし・v1.0のまま)。RLS/role入口判定でstaffロールは403
- ツンくま姉さんの指導メッセージは数値を本人に提示せず「行動」のみ指示する文面に限定(例:「HC提案、2回目のお客様には当日中に」○ /「あなたの成約率は23%」✕)

## 4. 設計判断の記録(なぜ2指標か)

売上と指名は①本人の行動と直接つながり②操作不能(スコアハックの余地がない)③報酬(歩合5%+指名¥250)と一致する。継続率・提案成功率を見せると「数字のための接客」(早すぎる提案・押し)が生まれ、visit_scoreを報酬に使わない原則(実装仕様書3-2)と同じ理由で表示自体を遮断する。**学習が知っていればよく、人が背負う必要のない数字は見せない。**
